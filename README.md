## 项目如何管理&开发

1. 从开始就集成，不采用分模块开发最后集成的做法
2. 多讨论，相互之间的技术交流促进进步
3. 编程在软件生命周期中占10-20%工作量
4. 设计错误63%，编码错误37%，发现越晚成本越高
5. 用正式文档资料把测试计划、详细测试方案以及实际测试结果保存下来，作为软件配置的一部分
6. 实现模块化设计的重要指导思想是：
   1. 分解
   2. 信息隐藏
   3. 模块独立性
7. 内聚更重要，应该把更多注意力集中到提高模块的内聚程度上
8. 高层模块应有较高的扇出，低层模块特别是底层模块应有较高的扇入
